HTML Forms Research

HTML Forms are very important to web design. According to “Your First HTML Form”, HTML forms play the role as one of the main elements that provide interaction between users and a website or application. They allow a website to receive data from users and most the time that data is sent to the web server, however, the web page could also intercept it to use on its own (“Your First HTML Form”).

A structure of an HTML form consists of one or more widgets which could be text fields in single line or multiline, select boxes, buttons, checkboxes, or radio buttons (“Your First HTML Form”). Those widgets always come with a label that describes their purpose most of the time (“Your First HTML Form”).

Like other HTML elements, HTML could also be styled by CSS. “Styling HTML Forms” states that it could be a little tricky while getting HTML Forms to work together with CSS. However, with the advantage of modern browsers, web designers now have more and more control over the design of form elements (“Styling HTML Forms”). There are good and bad things about using CSS with forms. The good thing is that some form elements can worked with CSS with very few if any problems across platforms, these include: form, fieldset, label and output (“Styling HTML Forms”). On the other hand, there are a couple of elements that could be rarely styled such as legend, checkboxes, radio buttons and placeholder (“Styling HTML Forms”). Moreover, things could be even worse since some elements simply could not be styled at all using CSS, which includes select, option, optgroup, datalist, progress and meter (“Styling HTML Forms”). One of the most important things to keep in mind while doing styling HTML Forms with CSS is that the user-agent style sheet of every browser can be a little inconsistent that requires few tricks to resolve (Styling HTML Forms).
